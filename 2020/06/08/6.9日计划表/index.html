后台管理系统样板

模式的话：一个页面最基本的还是要三部分构成

  1. 头顶需要有面包屑
  2. 左侧需要有到导航菜单
  3. 底部需要有分页

中间是一个表格、表格需要有表头、表头需要有颜色控制。

我们来看下实际的代码

                    async,await优化异步操作
            用了await就会得到一个数据对象，从这个数据对象身上解构赋值出一个data属性，之后重命名为res
            之后将打印res，将数据保存在data上。如下图所示，从对象身上拿到的数据，username,registe_time,city三个值的属性，将这三个值的属性绑定到tableData身上。那个这iteam就应该是重命名后的对象。
            到目前为止还没有渲染，渲染是最后一步的工作。
            async getUsers(){
                const Users = await getUserList({offset: this.offset, limit: this.limit});
                this.tableData = [];
                Users.forEach(item => {
                    const tableData = {};
                    tableData.username = item.username;
                    tableData.registe_time = item.registe_time;
                    tableData.city = item.city;
                    this.tableData.push(tableData);
                })
            }
        },

<template>
    <div class="header_container">

        <el-breadcrumb separator="/">
            <el-breadcrumb-item :to="{ path: '/manage' }">首页</el-breadcrumb-item>
            <el-breadcrumb-item v-for="(item, index) in $route.meta" :key="index">{{item}}</el-breadcrumb-item>
        </el-breadcrumb>
        <el-dropdown @command="handleCommand" menu-align='start'>
            <img :src="baseImgPath + adminInfo.avatar" class="avator">
            <el-dropdown-menu slot="dropdown">
                <el-dropdown-item command="home">首页</el-dropdown-item>
                <el-dropdown-item command="signout">退出</el-dropdown-item>
            </el-dropdown-menu>
        </el-dropdown>
    </div>
</template>

<script>
    import {signout} from '@/api/getData'
    import {baseImgPath} from '@/config/env'
    import {mapActions, mapState} from 'vuex'
    export default {
        data(){
            return {
                baseImgPath,
            }
        },
        created(){
            if (!this.adminInfo.id) {
                this.getAdminData()
            }
        },
        computed: {
            ...mapState(['adminInfo']),
        },
        methods: {
            ...mapActions(['getAdminData']),
            async handleCommand(command) {
                if (command == 'home') {
                    this.$router.push('/manage');
                }else if(command == 'signout'){
                    const res = await signout()
                    if (res.status == 1) {
                        this.$message({
                            type: 'success',
                            message: '退出成功'
                        });
                        this.$router.push('/');
                    }else{
                        this.$message({
                            type: 'error',
                            message: res.message
                        });
                    }
                }
            },
        }
    }
</script>

如上图所示：先拿到API文档的数据再进行渲染。

但是一般情况下是先进行假渲染，先绘制表格的样式，之后在方法里写数据。

这里需要和后端人员协调，协调个锤子，你英语有人家后端好么你去协调

以上就是获取数据表格的大致思路；

但是有一点：状态和操作这种需要作用域插槽进行渲染的；

模板slot-scope,scope用来接收数据，scope.row代表了当前列的数据。

提示：作用域插槽将会覆盖prop，作用域插槽渲染不需要涉及方法
